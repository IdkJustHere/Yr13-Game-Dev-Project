Project Overview

The 16-Week Roadmap

This plan builds Conveyor Kitchen incrementally across four four-week sprints. The strategy is: make it work, then make it look good, then make it feel good. Placeholder coloured rectangles are perfectly acceptable for the first half of the project.

Sprint 1 — Weeks 1–4

01
Foundation — Grid, Tiles & Item Flow
Weeks 1–4  |  Goal: Items visibly moving along placed conveyors
By the end of this sprint, you can place conveyor tiles on a grid, press Space to start, and watch coloured cubes slide from a Source to a Sink.

Week 1 — Project Setup & Grid Foundation

Autoload: GameManager

Create autoloads/GameManager.gd and register it as an Autoload singleton in Project Settings. This is the central nervous system of the game — it holds global state.

It should store: current game phase (PLANNING / RUSH_HOUR), the tick timer, current level config, inventory counts per tile type, and delivery quota progress. See Appendix A.

🎬 scenes/GameWorld.tscn Root: Node3D

The master scene. Everything lives under this node. Set it as the Main Scene in Project Settings.

GameWorld : Node3D Camera3D : Camera3D — Isometric cam, set projection to Orthogonal DirectionalLight3D : DirectionalLight3D — Scene lighting GridManager : Node3D — Instance of GridManager.tscn HUD : CanvasLayer — Instance of HUD.tscn (placeholder for now)
🗂 scenes/GridManager.tscn Root: Node3D

Manages the logical grid (a Dictionary mapping Vector2i cell coordinates to tile data) and renders floor tiles. This scene has no child nodes of its own — tiles and conveyors are spawned into it at runtime.

Attach scripts/GridManager.gd. The script holds the grid Dictionary, spawns floor tile MeshInstances at startup, and exposes functions: place_tile(cell, tile_type, rotation), remove_tile(cell), get_tile(cell). See Appendix B.

⚠️
Godot's 3D world uses X/Z as the horizontal plane (Y is up). Your grid cells map to Vector2i(x, z). When converting a grid cell to a world position, use Vector3(cell.x, 0, cell.y). This trips up almost everyone at first.
🎮 scenes/PlacementController.tscn Root: Node3D

Handles all mouse input for placing and removing tiles. Contains a GhostTile child that follows the cursor.

PlacementController : Node3D GhostTile : MeshInstance3D — Translucent box (cyan, alpha ~0.5) follows cursor Raycast : RayCast3D — Shoots ray from camera to mouse position

In _input(): cast a ray from the camera through the mouse position. If it hits the floor plane (Y=0), snap the hit point to the nearest grid cell and move GhostTile there. LMB calls GridManager.place_tile(), RMB calls GridManager.remove_tile(). R key rotates current selection. 1/2/3 keys change selected tile type. See Appendix C.

⚠️
Raycasting to an infinite plane requires using a Plane object and plane.intersects_ray() rather than relying on a physics collision body. See Appendix C for the pattern.

Add PlacementController as a child of GameWorld. Add a temporary flat MeshInstance3D (box mesh, scale 10×0.1×10) as the floor collision surface. Place a StaticBody3D + CollisionShape3D underneath it so the raycast has something to hit.

Week 2 — Conveyor Tiles & Direction Logic

🔧 scenes/ConveyorTile.tscn Root: Node3D

A single conveyor tile. For now, use a BoxMesh coloured dark grey with a small arrow decal (or just rotate it to imply direction).

ConveyorTile : Node3D Mesh : MeshInstance3D — BoxMesh placeholder (grey) ArrowMesh : MeshInstance3D — Tiny elongated box pointing in tile's direction

The script stores: tile_type (STRAIGHT / CORNER_L / CORNER_R), facing (0–3, representing N/E/S/W), and a function get_output_direction(input_dir) that returns which direction an item exits this tile given the direction it entered. See Appendix D for the direction logic table.

⚠️
Direction logic is the trickiest part of this sprint. A straight tile pointing North: item enters from South → exits North. A left corner pointing North: item enters from South → exits West. Build a lookup table (Dictionary of Dictionary) and test it manually before connecting it to item movement.

Week 3 — Sources, Sinks & Items

📦 scenes/SourceNode.tscn Root: Node3D

Spawns items. Colour-coded by item type: green = meat source, blue = fruit, yellow = spice. Placeholder: a tall BoxMesh.

SourceNode : Node3D Mesh : MeshInstance3D — Tall box, material colour = item type colour SpawnPoint : Marker3D — World position where items appear

Script exports: item_type (MEAT/FRUIT/SPICE), output_direction (which direction it ejects items). On each tick signal from GameManager, it calls spawn_item(). See Appendix E.

🎯 scenes/SinkNode.tscn Root: Node3D

Consumes matching items. Placeholder: a flat BoxMesh with a red-trimmed material.

SinkNode : Node3D Mesh : MeshInstance3D — Flat box, red border material

Script exports: accepted_item_type. Has function receive_item(item): if item type matches, emit item_delivered signal and destroy the item. See Appendix E.

🟥 scenes/ItemCrate.tscn Root: Node3D

A moving item. Placeholder: a small BoxMesh coloured by type (red/blue/yellow).

ItemCrate : Node3D Mesh : MeshInstance3D — Small box, colour = item type

Script stores: item_type, current_cell (Vector2i), current_direction. Has a move_to_cell(next_cell) function that tweens the world position over 0.4s (leaving 0.1s gap before the next tick). See Appendix F.

⚠️
Use Godot's built-in Tween (created via create_tween()) for smooth movement. Do NOT use _process() with manual lerp for this — it creates timing drift that breaks jam detection.

Week 4 — Tick System & Item Movement Integration

Now wire everything together through the GameManager tick system.

In GameManager.gd, add a Timer node (or use create_timer()) that fires every 0.5 seconds during Rush Hour. On each tick: call each Source's spawn logic, then call each live Item's advance logic (look up the tile at the item's current cell, get output direction, move to next cell). See Appendix A for the tick loop.

Game Systems — Jams, Scoring, UI & Levels
Weeks 5–8  |  Goal: All 5 campaign levels playable with win/lose states

Week 5 — Jam Detection & Fail States

Jam detection must run at the top of each tick, before any items move. The logic: build a Dictionary mapping each Vector2i cell to how many items will move into it this tick. Any cell with 2+ incoming items = Jam.

In GameManager.gd, add: jam_count (int, 0–3), MAX_JAMS = 3. On a Jam at a cell: emit jam_occurred(cell) signal, increment jam_count. If jam_count >= MAX_JAMS, emit kitchen_shutdown signal and end the run. See Appendix A.

Signals — Jam System

SignalEmitted byConnect to
jam_occurred(cell)GameManagerHUD._on_jam_occurred
jam_occurred(cell)GameManagerGridManager._on_jam_occurred (spawn particles)
kitchen_shutdownGameManagerGameWorld._on_kitchen_shutdown
level_timeoutGameManagerGameWorld._on_level_timeout

Jam clearing: in PlacementController.gd, during Rush Hour, if LMB is clicked on a jammed cell (check a jammed_cells Set in GameManager), call GameManager.clear_jam(cell) — this destroys one item in the jam, subtracts 10 points, and removes the cell from the jammed set. See Appendix C.

⚠️
Items that are part of a jam must be paused (do not advance them in subsequent ticks until the jam is cleared). Add a boolean is_jammed to ItemCrate and check it at the start of _on_tick().

Week 6 — Scoring & Timer

Add to GameManager.gd: score (int), deliveries (int), quota (int), time_remaining (float), tiles_used (int), tiles_available (int). Load these from a level config Resource (see below).

On each item delivery: score += 100, deliveries += 1. If deliveries reach quota, call level_complete(). The level timer counts down during Rush Hour only. On reaching zero, call level_timeout().

Grading in level_complete():
— Bronze: quota delivered before timer expires
— Silver: quota delivered with ≥30 seconds remaining
— Gold: Silver conditions AND tiles_used < tiles_available / 2

📄 scripts/LevelConfig.gd extends Resource

A custom Resource class (use class_name LevelConfig) storing all level parameters as exported variables: level_name, grid_size (Vector2i), quota, time_limit, tile_budget, tick_rate, obstacle_cells (Array[Vector2i]), source_configs (Array), sink_configs (Array). Save one .tres file per level. See Appendix G.

⚠️
Use a custom Resource (not a JSON file) for level data. This keeps everything in the Godot ecosystem and lets you edit values in the Inspector. Create levels/level_01.tres through level_05.tres.

Week 8 — Build All 5 Campaign Levels

Create five LevelConfig resource files (level_01.tres → level_05.tres). Each level is not a separate scene — GameWorld loads a LevelConfig and builds the level procedurally: spawning obstacles as MeshInstance3D boxes, placing SourceNode/SinkNode instances at configured positions.

Level Configurations

  • L1: 8×8, straight only, 1 source/sink, quota 10, 90s, 8 tiles
  • L2: 8×8, 1 corner required, pillar obstacle, quota 15, 90s, 10 tiles
  • L3 "Crossroads": 10×10, 2 sources/sinks crossing paths, quota 20, 120s, 14 tiles
  • L4: 10×10, freezer obstacle blocks, serpentine required, quota 20, 120s, 16 tiles
  • L5 "Gauntlet": 8×8, 3 item types, tick rate 0.3s, quota 25, 120s, 12 tiles

Camera Rotation (Mouse Wheel)

Add mouse wheel input to GameWorld.gd: cycle through 4 fixed camera transforms (NE/SE/SW/NW isometric angles). Store 4 preset Transform3D values and tween between them on scroll. See Appendix H.

⚠️
Isometric camera rotation in 3D is a Godot sticking point. The camera should orbit around the grid's centre point, not its own origin. Use a pivot Node3D at the grid centre; rotate the pivot and keep the camera as a fixed-offset child.

Week 9 — 3D Model Import & Conveyor Mesh

Source or create low-poly 3D models. Options: use Blender (free), purchase a low-poly kitchen asset pack, or use Godot's CSGBox + CSGCylinder nodes to assemble the conveyor from primitives without external tools.

Conveyor Tile Model (Straight)

The model needs: a flat 1m×1m base, and several roller cylinder meshes across it. If using Blender, export as .glb — Godot imports these natively. Place the model at assets/models/conveyor_straight.glb. Update ConveyorTile.tscn: remove the placeholder BoxMesh, add a child Node3D and instance the imported .glb scene inside it.

⚠️
When importing a .glb, Godot creates a read-only scene. To modify it (e.g., animate rollers), use "Advanced Import Settings" and enable "Use Named Skins" or simply create a new .tscn that instances the .glb and adds your modifications on top.

Conveyor Belt Animation

The rolling animation is visual only — it doesn't affect gameplay. In ConveyorTile.tscn, add an AnimationPlayer child. Create an animation called roll that rotates each roller cylinder around its local X-axis. Play this animation only during Rush Hour. Connect GameManager.phase_changed to a method that starts/stops the AnimationPlayer. See Appendix I.

Week 10 — Source, Sink, Item & Obstacle Models

Replace remaining placeholders:

  • SourceNode: Replace box with an industrial hopper model (or a CSG-assembled chute). Add a Label3D child showing the item type icon (emoji works for now: 🥩 🍎 🌶).
  • SinkNode: Replace box with a receptacle model. Add a red-accented trim material. A Label3D showing the accepted type.
  • ItemCrate: Replace box mesh with a slightly rounded cube. Add a subtle AnimationPlayer that plays a small bounce (scale Y from 1.0 to 1.1 to 1.0) on each tick arrival. See Appendix I.
  • Obstacles: Replace obstacle boxes with a freezer unit model (or a white CSG box with a blue tint material).
  • Floor Tiles: Replace the flat ground plane with individual 1m×1m tile meshes (white/checkerboard), one per cell, spawned by GridManager at startup.

Week 11 — Ghost Shader & Camera Polish

Ghost Placement Shader

Create assets/shaders/ghost_tile.gdshader. This shader renders the ghost tile as cyan, translucent, with a pulsing rim effect. Apply it as an override material on the GhostTile mesh. See Appendix J for the shader code.

⚠️
To make a material transparent in Godot 4, set Transparency to Alpha in the StandardMaterial3D, or use a shader with render_mode blend_mix. Without this, transparency won't render correctly.

Camera Tween Polish

Instead of snapping instantly between the 4 camera angles, tween the camera pivot's Y rotation over 0.3 seconds using create_tween().tween_property(). This requires the pivot Node3D approach from Sprint 2. Add a subtle camera sway: during Rush Hour, apply a very small sinusoidal offset to the camera's Y position (amplitude 0.05, frequency matching the tick rate) to give a "kitchen is alive" feel.

Week 12 — Visual Effects & UI Styling

Jam Particle Effect

Add a GPUParticles3D node to GameWorld.tscn called JamParticles. Configure it with a small spark burst (red particles, short lifetime, burst emission). On the jam_occurred(cell) signal: move JamParticles to the jammed cell's world position, call restart(). See Appendix K for particle settings.

Delivery Poof Effect

Similarly, add DeliveryParticles (white smoke puff, 8 particles, short lifetime) to GameWorld.tscn. Trigger it on item_delivered signal at the SinkNode's position.

UI Styling

Style the HUD and MainMenu using a Theme resource. Apply a clean industrial aesthetic: dark backgrounds, bright accent colours (use the item type colours), monospaced font for the timer. Import a free pixel or industrial font (e.g. from Google Fonts or itch.io) for the title. Apply the theme to MainMenu.tscn and HUD.tscn.

Style the ResultScreen: large grade label (GOLD in yellow, BRONZE in brown, FAILED in red). Add a subtle background panel with a dark translucent material.

04
Audio, Sandbox & Final Ship
Weeks 13–16  |  Goal: Shippable, polished, bug-free build

Week 13 — Audio System

🔊 autoloads/AudioManager.gd Autoload Singleton

A second autoload that manages all audio playback. This keeps audio calls clean — any script can call AudioManager.play_sfx("placement_thunk") without needing a reference to an AudioStreamPlayer node. See Appendix L.

AudioManager : Node SFXPlayer : AudioStreamPlayer — One-shot SFX playback MusicPlayer : AudioStreamPlayer — Looping ambient track TickPlayer : AudioStreamPlayer — Dedicated player for conveyor tick (needs precise timing)

Source free SFX from freesound.org or itch.io free packs. Required sounds per the GDD:

  • ui_click.wav — short mechanical button press
  • placement_thunk.wav — satisfying grid snap
  • conveyor_tick.wav — subtle rhythmic click
  • delivery_chime.wav — positive bell ring
  • jam_buzzer.wav — harsh error tone
  • ambient_loop.ogg — low industrial hum (must loop seamlessly)

Wire up calls: in PlacementController.gd, call AudioManager.play_sfx("placement_thunk") on tile placement. In SinkNode.gd, call delivery chime on receipt. In GameManager.gd, call jam buzzer on jam event. In GameWorld.gd or MainMenu.gd, start ambient loop on scene load.

⚠️
The conveyor tick sound must play in exact sync with the movement tick — even 50ms of drift feels wrong. Play TickPlayer at the same moment the tick signal fires in GameManager, not in response to a signal downstream.

Week 14 — Sandbox Mode

Sandbox Mode uses the same GameWorld scene but loads a special LevelConfig with: 20×20 grid, unlimited tiles (set tile_budget to 9999), 3 randomized Sources and Sinks, a 5-minute timer, and no quota (score attack instead).

Sandbox Configuration

Create levels/sandbox.tres with the 20×20 config. In GameManager.gd, add a is_sandbox flag. When true: no quota-based win condition; instead, the level ends when the timer hits zero. Final score is displayed on the ResultScreen.

Randomization: in GameWorld.gd, when loading sandbox config, use randi_range() to pick 3 random non-border, non-overlapping cells for Sources and 3 for Sinks. Assign item types randomly. Ensure each Source type has a matching Sink type. See Appendix M.

Unlock Sandbox: add a save_data.tres (another custom Resource) that tracks which levels are complete. Save it with ResourceSaver.save(). Sandbox unlocks when all 5 campaign levels are completed at any grade. Show a lock icon on the Main Menu Sandbox button until then.

⚠️
Godot's ResourceSaver / ResourceLoader is the right way to save game progress — not FileAccess with manual JSON. It's simpler and keeps everything in Godot's type system. See Appendix M.

Week 15 — Bug Fixing & Playtesting

This week is dedicated entirely to playtesting and fixing issues. Do not add new features. Run through the following systematic test pass:

  • Play every level from start to finish 3 times each
  • Intentionally create jams on every level — verify shutdown works
  • Achieve Gold on Level 1 — verify tile counting is accurate
  • Test all 4 camera angles on every level (can you see behind obstacles?)
  • Test pausing mid-Rush Hour, adjusting tiles, resuming
  • Complete all 5 levels → confirm Sandbox unlocks
  • Play Sandbox to timer end → verify score displays correctly
  • Click every UI button — no crashes or dead buttons
  • Quit and relaunch — confirm save data persists

Keep a bug log (a simple text file in the repo works). Prioritise: crashes first, wrong game logic second, visual issues third, audio issues last.

Week 16 — Final Polish & Export

Scrolling Arrow Texture on Belt

Per the GDD: a scrolling arrow texture on the conveyor surface indicating direction. In the conveyor's material (or shader), scroll the UV coordinates along the belt's facing axis every frame using TIME in a shader. See Appendix J for the UV scroll addition.

Export Settings

In Godot: Project → Export → Add a preset for your target platform (Windows/Mac/Web). For Web export (HTML5): enable "VRAM Texture Compression" for both BPTC (desktop) and ETC2 (mobile). Export to a folder. Test the exported build — some things behave differently outside the editor (input events, file paths, audio).

⚠️
Web (HTML5) export requires a server with Cross-Origin-Isolation headers for audio to work. The easiest way to test locally is with Godot's built-in "Export HTML5" and running via a local server. itch.io supports this natively.

# autoloads/GameManager.gd
# Register as Autoload singleton: Project Settings → Autoload
extends Node

enum Phase { PLANNING, RUSH_HOUR }
enum ItemType { MEAT, FRUIT, SPICE }

signal tick
signal phase_changed(new_phase)
signal jam_occurred(cell: Vector2i)
signal kitchen_shutdown
signal level_timeout
signal score_changed(new_score)
signal inventory_changed

var phase: Phase = Phase.PLANNING
var score: int = 0
var deliveries: int = 0
var quota: int = 20
var time_remaining: float = 90.0
var jam_count: int = 0
const MAX_JAMS = 3
var jammed_cells: Dictionary = {}
var tile_inventory: Dictionary = {}
var tiles_used: int = 0
var tiles_available: int = 12
var tick_rate: float = 0.5
var is_sandbox: bool = false

var _tick_timer: float = 0.0
var _all_items: Array = []

func _process(delta: float) -> void:
    if phase != Phase.RUSH_HOUR: return
    time_remaining -= delta
    if time_remaining <= 0.0:
        level_timeout.emit(); return
    _tick_timer += delta
    if _tick_timer >= tick_rate:
        _tick_timer -= tick_rate
        _do_tick()

func _do_tick() -> void:
    # 1. Check for would-be collisions BEFORE moving
    var next_positions: Dictionary = {}
    for item in _all_items:
        if item.is_jammed: continue
        var next = item.get_next_cell()
        if next in next_positions:
            next_positions[next].append(item)
        else: next_positions[next] = [item]
    # 2. Flag jams
    for cell in next_positions:
        if next_positions[cell].size() > 1:
            _trigger_jam(cell, next_positions[cell])
    # 3. Emit tick
    tick.emit()

func _trigger_jam(cell: Vector2i, items: Array) -> void:
    for item in items: item.is_jammed = true
    jammed_cells[cell] = items
    jam_count += 1
    jam_occurred.emit(cell)
    if jam_count >= MAX_JAMS: kitchen_shutdown.emit()

func toggle_phase() -> void:
    phase = Phase.RUSH_HOUR if phase == Phase.PLANNING else Phase.PLANNING
    phase_changed.emit(phase)

func clear_jam(cell: Vector2i) -> void:
    if not cell in jammed_cells: return
    var items = jammed_cells[cell]
    if items.size() > 0:
        items[0].queue_free()
        items.pop_front()
    if items.size() <= 1:
        jammed_cells.erase(cell)
        if items.size() == 1: items[0].is_jammed = false
    score -= 10; score_changed.emit(score)

func _on_item_delivered() -> void:
    deliveries += 1; score += 100; score_changed.emit(score)
    if not is_sandbox and deliveries >= quota: _level_complete()
Appendix B — GridManager.gd
# scripts/GridManager.gd
extends Node3D

var grid: Dictionary = {}
var obstacles: Dictionary = {}
var grid_size: Vector2i = Vector2i(8, 8)
const TILE_SIZE: float = 1.0

func cell_to_world(cell: Vector2i) -> Vector3:
    return Vector3(cell.x * TILE_SIZE, 0.0, cell.y * TILE_SIZE)

func world_to_cell(world_pos: Vector3) -> Vector2i:
    return Vector2i(roundi(world_pos.x / TILE_SIZE), roundi(world_pos.z / TILE_SIZE))

func is_valid_cell(cell: Vector2i) -> bool:
    return cell.x >= 0 and cell.x < grid_size.x \
       and cell.y >= 0 and cell.y < grid_size.y \
       and not cell in obstacles

func place_tile(cell: Vector2i, tile_scene: PackedScene, rotation_index: int) -> bool:
    if not is_valid_cell(cell) or cell in grid: return false
    var tile = tile_scene.instantiate()
    add_child(tile)
    tile.position = cell_to_world(cell)
    tile.rotation_degrees.y = rotation_index * 90.0
    tile.facing = rotation_index
    grid[cell] = tile; return true

func remove_tile(cell: Vector2i) -> bool:
    if not cell in grid: return false
    grid[cell].queue_free(); grid.erase(cell); return true

func get_tile(cell: Vector2i) -> Node3D: return grid.get(cell, null)

Appendix D — ConveyorTile Direction Logic
# scripts/ConveyorTile.gd
# Directions: 0=North(−Z), 1=East(+X), 2=South(+Z), 3=West(−X)
enum TileType { STRAIGHT, CORNER_L, CORNER_R }
enum Dir { NORTH, EAST, SOUTH, WEST }
@export var tile_type: TileType = TileType.STRAIGHT
var facing: int = 0

func get_output_direction(entry_dir: int) -> int:
    var relative_entry = (entry_dir - facing + 4) % 4
    var relative_exit: int = -1
    match tile_type:
        TileType.STRAIGHT:
            if relative_entry == 2: relative_exit = 0
            elif relative_entry == 0: relative_exit = 2
        TileType.CORNER_L:
            if relative_entry == 2: relative_exit = 3
            elif relative_entry == 1: relative_exit = 0
        TileType.CORNER_R:
            if relative_entry == 2: relative_exit = 1
            elif relative_entry == 3: relative_exit = 0
    if relative_exit == -1: return -1
    return (relative_exit + facing) % 4

func direction_to_delta(dir: int) -> Vector2i:
    match dir:
        Dir.NORTH: return Vector2i(0, -1)
        Dir.EAST:  return Vector2i(1,  0)
        Dir.SOUTH: return Vector2i(0,  1)
        Dir.WEST:  return Vector2i(-1, 0)
    return Vector2i.ZERO

Appendix F — ItemCrate.gd (Movement)
# scripts/ItemCrate.gd
extends Node3D
@export var item_type: GameManager.ItemType
var current_cell: Vector2i
var current_direction: int = 0
var is_jammed: bool = false

func get_next_cell() -> Vector2i:
    var grid_manager = get_tree().get_first_node_in_group("grid_manager")
    var tile = grid_manager.get_tile(current_cell)
    if not tile: return current_cell
    var out_dir = tile.get_output_direction(current_direction)
    if out_dir == -1: return current_cell
    return current_cell + tile.direction_to_delta(out_dir)

func _on_tick() -> void:
    if is_jammed: return
    var next_cell = get_next_cell()
    if next_cell == current_cell: return
    var grid_manager = get_tree().get_first_node_in_group("grid_manager")
    var sink = grid_manager.get_sink_at(next_cell)
    if sink:
        sink.receive_item(self); return
    var target_pos = grid_manager.cell_to_world(next_cell)
    current_cell = next_cell
    var tween = create_tween()
    tween.tween_property(self, "position", target_pos, 0.4)

## assets/shaders/ghost_tile.gdshader
shader_type spatial;
render_mode blend_mix;
uniform vec4 ghost_color : source_color = vec4(0.0, 0.9, 1.0, 0.45);
uniform float pulse_speed : hint_range(0.5, 5.0) = 2.0;
void fragment() {
    float pulse = sin(TIME * pulse_speed) * 0.15 + 0.85;
    ALBEDO = ghost_color.rgb; ALPHA = ghost_color.a * pulse;
}

shader_type spatial;
uniform sampler2D arrow_texture : source_color;
uniform float scroll_speed = 1.0;
void fragment() {
    vec2 scrolled_uv = UV + vec2(0.0, -TIME * scroll_speed);
    ALBEDO = texture(arrow_texture, scrolled_uv).rgb;
}

Appendix M — Sandbox Randomization & Save/Load
# In GameWorld.gd — sandbox source/sink randomization
func _place_sandbox_nodes() -> void:
    var used_cells: Array[Vector2i] = []
    var item_types = [GameManager.ItemType.MEAT, GameManager.ItemType.FRUIT, GameManager.ItemType.SPICE]
    item_types.shuffle()
    for i in range(3):
        var src_cell = _random_free_cell(used_cells); used_cells.append(src_cell)
        var snk_cell = _random_free_cell(used_cells); used_cells.append(snk_cell)
        _spawn_source(src_cell, item_types[i]); _spawn_sink(snk_cell, item_types[i])

func _random_free_cell(exclude: Array) -> Vector2i:
    var size = GameManager.current_config.grid_size
    while true:
        var cell = Vector2i(randi_range(1,size.x-2), randi_range(1,size.y-2))
        if not cell in exclude: return cell
